Basic Docker

Dec 23, 2015
1 minute read

I just discovered Docker a few days ago. It is awsome! Why am I excited about it? Because it provides a simple and easy means to play with lots of different environments and languages.

For istance, On my dev box I have a LAMP stack set up, which I use for main work and should keep in good shape. Now, I would like to play with other servers besides Apache. I want to experiment with NGINX, HAProxy, lighttpd, OpenLiteSpeed, etc. How would I do it? Mess up my LAMP trying to configure these servers to co-exist? Not easy/wise. Get a VPS and do the experiments there? Not cheap/convevient. The answer is “Use Docker!”

What Docker offers someone like me is basically easy-to-get, disposable environments to experiment with and learn new skills. For a lot of tech stacks there already is an image prepared by someone that you can pull and start using. If you need something special and you cannot find an image for, you can easily create one yourself.

This is the learning diary of a guy from the middle of nowhere.
